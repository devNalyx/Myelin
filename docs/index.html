What it is

A local daemon that watches your agentic coding sessions, notices recurring or high-stakes procedures, and — once a pattern earns it — crystallizes it into a real Claude Code Skill that keeps mutating from how it's actually used afterward.

It isn't "index my codebase" — that's NexusContext, the sibling project whose daemon/MCP/storage architecture this one is built on. Myelin's job is narrower and more personal: notice what you keep doing, and eventually stop making you re-explain it.

How a skill gets made

1. Observe

An agent reports a noteworthy procedure — a fix, a convention, a workaround — as it happens.

2. Warm up

Similar observations accumulate reps in a queue. Nothing hardens on the first sighting alone.

3. Promote

Enough reps, or an explicit high-stakes signal, and a real SKILL.md gets drafted and written — live immediately.

4. Stay alive

Corrections get appended straight into the live file. Confirmations build confidence. Unused skills get flagged, not forgotten.

Status

The full loop — observation → warmup queue → promotion → living SKILL.md → feedback → usage tracking — runs end to end today, with an optional embeddings-based similarity upgrade over the default token-overlap matching. Automatic session-transcript ingestion (with a redaction pass ahead of it) is in active development. See the README for the honest, current split between what's implemented and what's still sketch.
